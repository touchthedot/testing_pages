Использование OpenColorIO

Дополнительно к поддержке редактирования 32-битных изображений Affinity Designer также поддерживает OpenColorIO — систему управления цветом, благодаря которой можно обеспечить рабочий процесс с полным управлением цветами. Она используется преимущественно для производства кинофильмов, но может быть использована в любой ситуации, когда требуется точное управление цветами от начала до конца.

Настройка OpenColorIO

По умолчанию функции OpenColorIO в Designer недоступны для немедленного использования. Для этого требуется файл конфигурации .ocio вместе с некоторыми вспомогательными файлами, например таблицами перекодировки.

На веб-сайте OpenColorIO (http://www.opencolorio.org) представлены некоторые образцы конфигурации с целым рядом подходящих профилей ввода и вывода, в том числе несколько конфигураций Academy Colour (ACES).

Чтобы настроить конфигурацию OpenColorIO, выполните приведенные далее действия.
  1. Загрузите и извлеките выбранную конфигурацию OpenColorIO в папку.
  2. В меню Affinity Designer выберите Установки.
  3. В меню Правка выберите пункт Установки.
  4. На панели Цвет в пункте Файл конфигурации OpenColorIO нажмите Выбрать и перейдите к каталогу с извлеченной конфигурацией. Выберите файл конфигурации .ocio.
  5. В пункте Папка поиска OpenColorIO нажмите Выбрать и выберите каталог с извлеченной конфигурацией (должен отобразиться уже выделенный каталог, из которого был выбран файл конфигурации .ocio).
  6. Отобразится запрос на перезагрузку программы, необходимую для вступления параметров OpenColorIO в силу.

Использование OpenColorIO

Отображение функции OpenColorIO можно включить двумя способами.

  • На панели 32-битный просмотр содержится параметр Трансформация отображения, который становится доступным только при наличии действительной конфигурации OpenColorIO. Его можно использовать для обеспечения неразрушающего рабочего процесса управления цветами. Дополнительную информацию см. в разделе 32-битный просмотр.
  • Можно добавить корректирующий слой OCIO (см. Корректировка OCIO), чтобы без потерь преобразовывать цветовые пространства. В документе можно создать несколько корректирующих слоев OCIO, что обеспечивает возможность размещать составные слои из разных цветовых пространств. Пример стека слоев (в иерархическом порядке) приведен далее.
    • Корректировка OCIO — из Утилита — линейное — sRGB обратно в Роль — scene_linear
    • Пиксельный слой sRGB — составной элемент.
    • Корректировка OCIO — из Роль — scene_linear в Утилита — линейное — sRGB
    • Пиксельный слой — исходный слой.

При загрузке документов OpenEXR Designer всегда преобразовывает исходное цветовое пространство в scene_linear. При наличии действительной конфигурации OpenColorIO Designer также отобразит сообщение о том, какой цветовой профиль был преобразован. Обычно это можно определить по аффиксу имени файла, например render_acescg.exr.
